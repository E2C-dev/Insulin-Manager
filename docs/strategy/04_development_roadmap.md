## Phase 1（2026年3月〜5月）— 「最初のユーザーを獲得し、価値を届ける」

### 目標

| 指標 | 目標値 |
|-----|--------|
| 累積ユーザー数 | 3,500人 |
| 有料ユーザー数 | 200人 |
| SNSフォロワー合計 | 1,000人 |
| NPS | +20以上 |

### 開発タスク

#### 1-1. 妊娠糖尿病専用オンボーディングフロー

**優先度: 最高 / 期限: 2026年3月末**

現状の課題:
- 新規ユーザーが最初の設定（インスリンプリセット・計算ルール）で離脱するケースがある
- 妊娠糖尿病患者に特化した導線が存在しない

実装内容:
- [ ] ユーザー登録時の「糖尿病タイプ選択」画面の追加（妊娠糖尿病 / 1型 / 2型 / その他）
- [ ] 妊娠糖尿病選択時の専用オンボーディングフロー（5ステップ）
  - Step 1: インスリンの種類入力（超速効型・中間型・混合型）
  - Step 2: 担当医から指示された目標血糖値の入力
  - Step 3: 自動計算ルールの初期設定ウィザード
  - Step 4: 最初の記録をガイド付きで入力
  - Step 5: 完了画面 + 次のアクションの提示
- [ ] オンボーディング完了率の計測実装（Analytics）
- [ ] オンボーディング中断ユーザーへのプッシュ通知リマインダー

**技術仕様**:
- 既存のオンボーディングコンポーネントを拡張
- タイプ別フロー分岐をルーティングレベルで制御
- 設定データはユーザープロフィールに保存

---

#### 1-2. ランディングページ（LP）の更新

**優先度: 最高 / 期限: 2026年3月末**

現状の課題:
- 現在のLPが機能説明中心で、患者の感情・課題への共感が薄い
- 妊娠糖尿病・1型・2型それぞれへの訴求が統一されていない

実装内容:
- [ ] ファーストビューのコピー刷新（「血糖値に基づく自動計算」を前面に）
- [ ] 開発背景ストーリーセクションの追加（妻の妊娠糖尿病エピソード）
- [ ] セグメント別ユースケースセクション（妊娠糖尿病 / 1型 / 2型）
- [ ] ユーザー体験談セクション（テキスト + スクリーンショット）
- [ ] プラン比較テーブルの見直し（無料・スタンダード・プレミアムの違いを明確化）
- [ ] CTAボタン最適化（A/Bテスト設計）

---

#### 1-3. 記録UXの改善

**優先度: 高 / 期限: 2026年4月末**

実装内容:
- [ ] 血糖値入力 → インスリン量自動計算 → 記録の3ステップをワンフローに統合
- [ ] 記録画面にショートカット（「食前」「食後」「就寝前」のクイック選択）
- [ ] 記録完了後の次回記録リマインダー設定UI
- [ ] グラフ表示の改善（7日間・30日間の推移グラフ）

---

#### 1-4. SNS・GTM立ち上げ

**優先度: 高 / 期限: 2026年3月中旬**

実装内容:
- [ ] Instagram公式アカウント開設・プロフィール整備
- [ ] X（旧Twitter）公式アカウント開設・プロフィール整備

#### 2-1. Stripeによる課金実装

**優先度: 最高 / 期限: 2026年6月末**

実装内容:
- [ ] Stripe Billing の導入（サブスクリプション管理）
- [ ] スタンダードプラン（480円/月）の課金フロー実装
- [ ] プレミアムプラン（980円/月）の課金フロー実装
- [ ] 年払いオプションの検討・実装（スタンダード4,800円/年 = 2ヶ月無料）

- [ ] 請求書・領収書の自動発行

#### 2-2. 機能制限（フリーミアム）実装

**優先度: 最高 / 期限: 2026年7月末**

| 機能 | 無料 | スタンダード | プレミアム |

| 自動計算ルール | 1ルールのみ | 最大5ルール | 無制限 |
| インスリンプリセット | 3種類まで | 無制限 | 無制限 |
| グラフ・統計 | 7日間 | 90日間 | 無制限 |
| CSV/PDFエクスポート | - | あり | あり |
| プッシュ通知リマインダー | 基本のみ | カスタム設定 | カスタム設定 |
| CGM連携（Phase 3以降） | - | - | あり |
| 管理栄養士相談（Phase 4） | - | - | あり |
| 優先サポート | - | - | あり |

実装内容:
- [ ] ユーザープランの判定ロジック実装（API・フロントエンド両方）
- [ ] 制限到達時のアップグレード訴求UI（モーダル・バナー）
- [ ] 「あと〇日で制限に達します」のプリエンプティブ通知
- [ ] A/Bテスト：制限到達UXの最適化

## Phase 3（2026年9月〜11月）— 「コア差別化機能を拡張する」

### 目標

| 指標 | 目標値 |
|-----|--------|
| 累積ユーザー数 | 50,000人 |
| 有料ユーザー数 | 5,000人 |
| MRR | 3,000,000円以上 |
| プレミアム比率 | 30%以上 |

### 開発タスク

#### 3-1. CGM連携（FreeStyle リブレ）

**優先度: 最高 / 期限: 2026年9月末**

背景: CGM（持続血糖測定器）ユーザーは手動入力の手間がなくなり、より精密な自動計算が可能になる。プレミアムプランの最大の差別化機能。

実装内容:
- [ ] Abbott LibreLink APIの調査・連携申請
- [ ] LibreLinkUp APIによるデータ取得実装
- [ ] CGMデータ（5分間隔）の保存・表示
- [ ] CGMデータを使ったインスリン計算ルールの拡張
  - 現在値だけでなく「上昇トレンド/下降トレンド」を考慮した計算
- [ ] リアルタイム血糖値ウィジェット（ホーム画面）
- [ ] CGMデータと手動記録の統合表示
- [ ] 高血糖・低血糖アラート通知

**技術仕様**:
- LibreLink Up API（非公式APIの調査も含む）
- バックグラウンド同期（15分間隔）
- データ保存: 既存DBスキーマの拡張

---

#### 3-2. アフィリエイト機能（宅配食連携）

**優先度: 高 / 期限: 2026年10月末**

背景: 糖尿病患者向け宅配食サービスとのアフィリエイト連携により、収益の多様化を図る。

対象アフィリエイトパートナー候補:
- nosh（nosh-ナッシュ）
- ベルーナグルメ（糖質制限食）
- 三ツ星ファーム
- ヨシケイ

実装内容:
- [ ] アフィリエイトパートナーとの契約・リンク発行
- [ ] アプリ内「おすすめ食事」セクションの追加
  - 血糖値パターンに基づくパーソナライズド提案（プレミアム機能）
  - 宅配食サービスへのアフィリエイトリンク
- [ ] クリック・購入のコンバージョン計測実装
- [ ] アフィリエイト収益レポートの管理画面

---

#### 3-3. データエクスポート・医師共有機能

**優先度: 中 / 期限: 2026年11月末**

実装内容:
- [ ] 記録データのPDFレポート出力（1ページサマリー + 詳細）
- [ ] CSV/Excel形式のエクスポート
- [ ] 「医師に共有」機能（URLベースの閲覧専用リンク生成）
- [ ] 次回受診日リマインダー

---

### Phase 3 完了定義

- [ ] アフィリエイト機能リリース・初月収益発生

## Phase 4（2026年12月〜2027年2月）— 「プラットフォーム化・B2B展開」

### 目標

| 指標 | 目標値 |
|-----|--------|
| 累積ユーザー数 | 130,000人 |
| 有料ユーザー数 | 17,000人 |
| MRR | 10,000,000円以上 |
| B2B導入クリニック数 | 5施設以上 |

### 開発タスク

#### 4-1. 管理栄養士マッチング機能

**優先度: 最高 / 期限: 2027年1月末**

背景: プレミアムユーザーへの付加価値として、管理栄養士とのリモート相談機能を提供。血糖値データをベースにした食事指導を可能にする。

実装内容:
- [ ] 管理栄養士プロフィール登録・審査フロー
- [ ] ユーザー↔管理栄養士のマッチングアルゴリズム（専門領域・対応可能時間）
- [ ] チャット機能（テキスト + 記録データ共有）
- [ ] ビデオ相談機能（外部SDK連携を検討: Twilio / Agora）
- [ ] 相談予約・カレンダー機能
- [ ] 管理栄養士への報酬支払いフロー（Stripe Connect）
- [ ] 相談履歴の保存・次回予約への引き継ぎ

**収益モデル案**:
- プレミアムユーザー: 月1回の相談を含む（管理栄養士への報酬はインスリアが負担）
- 追加相談: 1回2,000円（アプリ内課金）

---

#### 4-2. B2B機能（医療機関向け）

**優先度: 高 / 期限: 2027年2月末**

背景: 医師・医療機関がインスリアを患者に推奨し、患者データを診療に活用できるプラットフォームを構築。

実装内容:
- [ ] 医療機関向け管理ダッシュボード
  - 患者一覧表示
  - 患者別血糖値推移・投与記録の閲覧
  - アラート（低血糖・高血糖の頻発患者の自動検知）
- [ ] 患者↔医師のデータ共有同意フロー（明示的な同意取得）
- [ ] クリニック向けプランの設計（施設契約・複数患者管理）
- [ ] 電子カルテ連携の調査（SS-MIX2対応の検討）
- [ ] HIPAA/個人情報保護法対応の確認・強化

**B2B料金モデル案**:
- クリニックプラン: 月額9,800円（患者50名まで）
- エンタープライズ: 要問合せ

---

#### 4-3. プラットフォームのスケーラビリティ強化

**優先度: 中 / 期限: 通期**

実装内容:
- [ ] データベースの最適化（インデックス・クエリ最適化）
- [ ] APIのレートリミット・キャッシュ実装
- [ ] エラーモニタリング強化（Sentry等の導入）
- [ ] ロードテスト実施（10万ユーザー規模）
- [ ] セキュリティ監査（医療データ取り扱い）
